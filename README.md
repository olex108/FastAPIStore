# СЕРВИС ПОКУПКИ ТОВАРОВ ДЛЯ АВТОРИЗИРОВАННЫХ ПОЛЬЗОВАТЕЛЕЙ

API backend приложение на Python. Сервис позволяет пользователям регистрироваться, авторизоваться и просматривать список доступных товаров. Авторизованные пользователи могут добавлять товары в корзину и управлять ею.

## СТЕК ТЕХНОЛОГИЙ

* **Python 3.13 / FastAPI**
* **PostgreSQL / SQLAlchemy**
* **Redis** (Брокер)
* **TaskiQ** (Задачи по расписанию)
* **Docker / Docker Compose** (Контейнеризация)


## БЫСТРЫЙ СТАРТ (ЛОКАЛЬНО)

### Установить
* [Docker](https://www.docker.com)
* [Docker Compose](https://docs.docker.com)

### Настроить

1. Клонируйте репозиторий:

   ```bash
   git clone https://github.com/olex108/FastAPIStore.git
   ```
   
2. Настройте файл .env в корне проекта по примеру .env.example, а также .ent.test для тестирования приложения:

    ```bash
    # Main settings
    HOST=host#127.0.0.1
    PORT=port#8000
    SECRET_KEY=secret_key
    DEBUG=True
    # postgres DB
    DB_USERNAME=postgres
    DB_PASSWORD=password
    DB_HOST=host
    DB_PORT=port
    DB_NAME=name
    # SMTP
    SMTP_HOST=://gmail.com
    SMTP_PORT=465
    SMTP_USER=your_email@gmail.com
    SMTP_PASS=your_app_password
    # Redis
    REDIS_HOST=host
    REDIS_PORT=port
   ```

3. Создать и запустить приложение:

    ```bash
    docker-compose up --build
    ```

Команда создаст приложение, настроит подключение к базе данных и проведет миграции, создаст и подключит брокер задач. Для проверки работы приложения http://localhost:8080/docs#


4. Проверка работоспособности

При создании и запуске приложения проводится автоматическое тестирование

5. Настройка и управление программой

Запуск кастомной программы загрузки ролей и разрешений для пользователей

```bash
docker exec -it fastapi_app python src/scripts/default_settings.py
```

В случае необходимости отката миграций
Удаление таблиц

```bash
docker exec -it store_db psql -U postgres -d fa_store_db -c "DELETE FROM alembic_version;"
```
Запуск миграций

```bash
docker exec -it fastapi_app alembic upgrade head
```

Просмотр таблиц

```bash
docker exec -it store_db psql -U postgres -d fa_store_db -c "\dt" 
```

## ОПИСАНИЕ ПРОЕКТА

#### Основной функционал 

1. **Регистрация пользователей:** 
   - Регистрации пользователей с полями: ФИО, email (уникальный), телефон (уникальный), пароль и подтверждение пароля. Все поля обязательны.
   - Пароль должен соответствовать следующим требованиям: не менее 8 символов, только латиница, минимум 1 символ верхнего регистра, минимум 1 спец символ ($%&!:).
   - Телефон должен начинаться с +7 и содержать 10 цифр.
2. **Авторизация пользователей:** 
   - Авторизации пользователей с использованием email или телефона и пароля.
   - После авторизации пользователь получает доступ к списку всех активных товаров в базе данных.
3. **Работа с товарами:** 
   - Создание таблицы товаров с полями: name (str), price (int), quantity (int), created_at (datetime), updated_at (datetime), is_active (bool).
   - Методы для добавления, редактирования и удаления товаров (доступны только администратору).
4. **Корзина:** 
   - Корзина содержит список товаров и их количество.
   - Методы для добавления товаров в корзину (одного и нескольких), удаления товаров из корзины и полной очистки корзины.
   - При просмотре корзины получение общей стоимости товаров в корзине.
5. **Доступ к API:** 
   - Методы регистрации и авторизации доступны всем пользователям без аутентификации.
   - Методы для работы с товарами и корзиной доступны только авторизованным пользователям.
   - При попытке неавторизованного доступа возвращать ошибку {code: 401, message: "Unauthorized"} с HTTP статусом 401.

#### Дополнения

1. **Работа с товарами. По одному запросу:** 
   - Пагинация списка товаров по указанному пользователем лимиту. Но в пределах ограничения от 1-50.
   - Фильтрация списка товаров по полю name (str).
   - Сортировка списка товаров по полям name (str) или price (int).
2**Корзина:** 
   - Отдельные эндпоинты для управления корзиной пользователя менеджером по заданию user_id.
   - Метод создания заказа по составленной корзине пользователем. Отправка письма с чеком на оплату заказа пользователю 
   - Метод для просмотра истории заказов.
3**Кастомные команды:** 
   - Команда для создания или сброса ролей(групп) и разрешений.
   - Команда для ручного управления добавления и удаления роли(группы) у пользователя.
   - Команда для ручного управления добавления и удаления разрешения у ролей(групп).

## ТЕХНИЧЕСКИЕ РЕШЕНИЯ:

 1. **База данных**: 
    - Создание и проведение миграций посредством библиотеки alembic.
    - При создании новых моделей обязательно делать импорт модели в /models/__init__.py
    - Все модели должны наследоваться от Base из /models/base.py
 
Команда создания миграций
```commandline
alembic revision --autogenerate -m "Initial commit"
```
Команда миграций в базу данных
```commandline
 alembic upgrade head
```

 2. **Контейнеризация**: 
    - Используется Docker и Docker-compose для контейнеризации приложения. Для тестов внутри контейнера обязательно в корне проекта создать .env.test
    - Контейнеры: app, db, test_db, redis, worker, pytest
    
 3. **Аутентификация и авторизация**: 

    - Система поддерживает двойной способ входа — пользователь может использовать email или номер телефона в качестве логина.
    - Используется стандарт JWT (JSON Web Token) с механизмом двух токенов:
Access Token: Короткоживущий токен для доступа к защищенным эндпоинтам (передается в заголовке Authorization: Bearer <token>).
Refresh Token: Долгоживущий токен для получения новых access-токенов (хранится в базе данных).
    - Управление сессиями: Refresh-токены являются stateful (хранятся в базе данных). Это позволяет серверу валидировать сессию при обновлении токена и, при необходимости, отозвать её.
 4. **Права доступа**:
    Защита ресурсов: Доступ к эндпоинтам ограничивается через зависимости
    ##### get_current_user:
    - Проверяет валидность и срок действия JWT.
    - Автоматически извлекает пользователя из БД вместе с его правами доступа (permissions) для последующей проверки полномочий.
    ##### get_current_active_user:
    - Проверяет является ли пользователь активным
    ##### PermissionChecker
    - Проверяет есть ли в списке разрешений пользователя запрашиваемые эндпоинтом
    ##### OwnerOrPermissionChecker
    - Проверяет является ли пользователь автором(владельцем) или есть ли в списке разрешений пользователя запрашиваемые эндпоинтом
 5. **Пароли**: 
    - Пароли должны соответствовать требованиям безопасности.
    - Реализована защита от атак по времени (метод dammy_verify): даже если пользователь не найден в базе, система выполняет имитацию проверки пароля, чтобы время ответа сервера всегда было одинаковым.
    - Применяется хширование паролей.
 6. **Тестирование**: 
  - Код покрыт тестами на 79%.
 7. **Асинхронность**: 
    Для обеспечения асинхронности используется фреймворки, технологии и библиотеки
    - FastAPI для создания асинхронных эндпоинтов
    - sqlalchemy[asyncio], asyncpg, psycopg2-binary для асинхронного подключения к базе данных
    - taskiq-fastapi для создания асинхронных задач
    - aiosmtplib для асинхронной отправки почты
